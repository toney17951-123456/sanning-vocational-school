# 三宁职校官网项目Sub Agent分配方案

## 一、Sub Agent角色与职责分配

### 1. 前端开发Agent
- **角色名称**：frontend-dev
- **主要职责**：负责所有前端功能开发
- **工作内容**：
  - 初始化Vue 3 + Vite项目
  - 开发前台页面（首页、学校简介、师资介绍、业务项目、新闻中心）
  - 开发在线报名与支付前端功能
  - 开发后台管理系统前端
  - 实现响应式设计
  - 与后端Agent进行联调

### 2. 后端开发Agent
- **角色名称**：backend-dev
- **主要职责**：负责所有后端功能开发
- **工作内容**：
  - 初始化Flask项目
  - 设计和实现数据库表结构
  - 开发RESTful API接口
  - 集成微信支付和支付宝API
  - 实现JWT认证和权限控制
  - 与前端Agent进行联调

### 3. 数据库管理Agent
- **角色名称**：db-admin
- **主要职责**：负责数据库设计、优化和维护
- **工作内容**：
  - 设计数据库表结构
  - 优化数据库查询性能
  - 制定数据备份与恢复策略
  - 协助后端Agent进行数据库操作

### 4. UI/UX设计Agent
- **角色名称**：ui-ux-designer
- **主要职责**：负责界面设计和用户体验
- **工作内容**：
  - 设计前台页面UI
  - 设计后台管理系统UI
  - 制定响应式设计规范
  - 提供设计稿和资源

### 5. 测试Agent
- **角色名称**：tester
- **主要职责**：负责功能测试、性能测试和安全性测试
- **工作内容**：
  - 编写测试用例
  - 进行功能测试
  - 进行响应式测试
  - 进行性能测试
  - 进行安全性测试
  - 提交bug报告

## 二、协作流程与命令规范

### 1. 项目初始化阶段

#### 前端开发Agent
```bash
# 初始化Vue 3 + Vite项目
npm create vite@latest frontend -- --template vue
cd frontend
npm install
npm install element-plus vue-router@4 pinia axios sass
# 配置项目结构
```

#### 后端开发Agent
```bash
# 初始化Flask项目
mkdir backend
cd backend
python -m venv venv
venv\Scripts\activate  # Windows
pip install flask flask-sqlalchemy flask-jwt-extended pymysql cryptography openpyxl pillow
# 配置项目结构
```

#### 数据库管理Agent
```bash
# 初始化MySQL数据库
mysql -u root -p
CREATE DATABASE saning_school CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'saning_user'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON saning_school.* TO 'saning_user'@'localhost';
FLUSH PRIVILEGES;
```

### 2. 开发阶段

#### 前端开发Agent
```bash
# 启动开发服务器
npm run dev
# 构建项目
npm run build
# 代码规范检查
npm run lint
```

#### 后端开发Agent
```bash
# 启动开发服务器
flask run --debug
# 运行数据库迁移
flask db migrate
flask db upgrade
# 运行单元测试
python -m pytest
```

#### 测试Agent
```bash
# 运行API测试
postman run api_test_collection.json
# 运行UI测试
python -m pytest ui_tests/
```

### 3. 部署阶段

#### 前端开发Agent
```bash
# 构建生产版本
npm run build
# 部署到服务器
scp -r dist/ user@server:/var/www/html/
```

#### 后端开发Agent
```bash
# 安装生产依赖
pip install gunicorn
# 启动Gunicorn服务器
gunicorn -w 4 -b 0.0.0.0:5000 app:app
# 配置Nginx
```

## 三、文档与资源共享

### 3. 代码仓库
- 前端代码：`./frontend/`
- 后端代码：`./backend/`
- 测试代码：`./tests/`

## 四、沟通与协作机制

### 1. 日常沟通
- 使用Trae IDE的聊天功能进行实时沟通
- 每日更新工作进度，记录在`./progress.md`文件中

### 2. 任务分配与跟踪
- 使用Trae IDE的TodoWrite工具创建和管理任务
- 每个Agent负责的任务在`./tasks/`目录下创建对应的任务文件

### 3. 代码评审
- 使用Trae IDE的Edit和MultiEdit工具进行代码编辑
- 代码变更前使用Read工具查看最新代码
- 重要代码变更需要其他Agent进行评审

## 五、开发规范

### 1. 前端开发规范
- 遵循Vue 3官方最佳实践
- 使用Element Plus组件库
- 代码风格：ESLint + Prettier
- 提交规范：Conventional Commits

### 2. 后端开发规范
- 遵循Flask官方最佳实践
- 使用RESTful API设计规范
- 代码风格：PEP 8
- 提交规范：Conventional Commits

### 3. 数据库开发规范
- 遵循MySQL最佳实践
- 使用InnoDB存储引擎
- 合理设计索引
- 定期进行数据备份

## 六、质量保证

### 1. 测试流程
- 每个功能模块开发完成后，由开发Agent进行单元测试
- 集成测试由测试Agent负责
- 系统测试由所有Agent共同参与

### 2. 性能要求
- 页面加载时间 < 2秒
- API响应时间 < 500ms
- 数据库查询时间 < 100ms

### 3. 安全要求
- 密码使用bcrypt加密存储
- API使用JWT认证
- 防止SQL注入、XSS和CSRF攻击
- 支付功能遵循安全规范

## 七、项目进度管理

### 1. 进度跟踪
- 每个Agent每日更新工作进度
- 每周召开项目例会，讨论进度和问题
- 进度报告存放在`./progress.md`文件中

### 2. 风险管理
- 识别潜在风险，记录在`./risks.md`文件中
- 制定风险应对措施
- 定期评估风险状态

## 九、启动命令

```bash
# 创建项目目录结构
mkdir -p frontend backend design tests .trae/documents

# 初始化前端项目
cd frontend
npm create vite@latest . -- --template vue
npm install
npm install element-plus vue-router@4 pinia axios sass

# 初始化后端项目
cd ../backend
python -m venv venv
venv\Scripts\activate  # Windows
pip install flask flask-sqlalchemy flask-jwt-extended pymysql cryptography openpyxl pillow

# 创建数据库
mysql -u root -p -e "CREATE DATABASE saning_school CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; CREATE USER 'saning_user'@'localhost' IDENTIFIED BY 'password'; GRANT ALL PRIVILEGES ON saning_school.* TO 'saning_user'@'localhost'; FLUSH PRIVILEGES;"
```

### 2. 启动开发服务器
```bash
# 启动前端开发服务器（在frontend目录下）
npm run dev

# 启动后端开发服务器（在backend目录下）
venv\Scripts\activate  # Windows
flask run --debug
```

## 十、注意事项

1. 所有Agent必须严格遵循开发规范和协作流程
2. 代码变更前必须先查看最新代码，避免冲突
3. 重要功能变更需要经过评审才能提交
4. 遇到问题及时沟通，共同解决
5. 定期更新工作进度，确保项目按时完成
