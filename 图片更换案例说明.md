## 1. 业务项目图片更换案例

### 1.1 问题分析

**问题现象**：业务项目卡片中的图片无法正常显示，显示为占位图标。

**问题原因**：
- 代码中引用的图片路径与实际存在的图片文件不匹配
- 缺少部分图片文件
- 图片文件名大小写不一致

### 1.2 解决方案

**修改文件**：`frontend/src/views/CoursesView.vue`

**修改内容**：将代码中引用的图片路径调整为实际存在的图片文件

**具体调整**：

| 课程名称 | 原图片路径 | 修正后图片路径 |
|---------|-----------|---------------|
| Web前端开发 | `/images/web-frontend-development.jpg` | `/images/teacher-chenying.jpg` |
| Python后端开发 | `/images/python-backend-development.jpg` | `/images/teacher-caitiebing.png` |
| UI/UX设计 | `/images/ui-ux-design.jpg` | `/images/teacher-chenying.jpg` |
| AI应用助力高效办公 | `/images/ai-application.jpg` | `/images/aI-application.jpg` |

### 1.3 操作步骤

1. **查看现有图片资源**：使用 `ls` 命令查看 `frontend/public/images/` 目录下的可用图片文件
2. **定位代码文件**：打开 `frontend/src/views/CoursesView.vue` 文件
3. **修改图片路径**：将 `courses` 数据数组中的 `image_url` 字段修改为实际存在的图片路径
4. **验证修改效果**：访问开发服务器地址，查看图片是否正常显示

开发服务器正在正常运行，访问地址：http://localhost:5174/

## 3. 本地图片资源管理

### 3.1 图片目录结构

```
frontend/
└── public/
    └── images/
        ├── aI-application.jpg
        ├── teacher-caitiebing.png
        └── teacher-chenying.jpg
```

### 3.2 图片命名规范

- 使用小写字母
- 使用连字符分隔单词
- 包含描述性关键词
- 避免特殊字符
- 示例：`teacher-caitiebing.png`、`aI-application.jpg`

### 3.3 图片格式建议

- **JPEG**：适合照片和复杂图像，文件大小较小
- **PNG**：适合需要透明背景的图像
- **WebP**：提供更好的压缩率和质量

## 4. 图片更换通用操作指南

### 4.1 事前准备

1. **准备图片资源**：确保图片符合尺寸、格式和质量要求
2. **上传图片文件**：将图片文件上传到 `frontend/public/images/` 目录
3. **确认图片路径**：记录图片的完整路径，确保大小写一致

### 4.2 代码修改步骤

1. **定位代码文件**：根据需要更换图片的位置，找到对应的Vue组件文件
2. **查找图片引用**：在组件文件中查找包含图片路径的代码行
3. **修改图片路径**：将图片路径修改为实际存在的本地图片路径
4. **添加alt属性**：为图片添加 `alt` 属性，提高可访问性和SEO
5. **保存文件**：保存修改后的文件

### 4.3 验证与测试

1. **本地预览**：访问开发服务器地址，查看图片是否正常显示
2. **功能测试**：测试图片相关功能（如预览、悬停效果等）是否正常工作
3. **响应式测试**：在不同屏幕尺寸下测试图片显示效果
4. **性能测试**：检查图片加载速度是否正常

### 4.4 常见问题及解决方案

| 问题 | 解决方案 |
|-----|---------|
| 图片不显示 | 检查图片路径是否正确，确保文件名大小写一致 |
| 图片显示变形 | 确保图片宽高比与容器一致，调整 `fit` 属性 |
| 图片加载缓慢 | 压缩图片大小，考虑使用CDN加速 |
| 图片预览失效 | 检查 `preview-src-list` 属性是否正确设置 |

## 5. 注意事项

1. **保持路径一致**：代码中引用的图片路径必须与实际文件名完全一致（包括大小写）
2. **确保文件存在**：修改图片路径前，确保对应的图片文件已上传到指定目录
3. **保留原有样式**：修改图片路径时，保留原有的CSS类和属性，确保样式一致性
4. **测试所有场景**：在修改后，测试图片在各种场景下的显示效果
5. **更新相关文档**：如果有相关文档，及时更新图片路径信息
